O QUE VOCÊ PRECISA ANTES DE COMEÇAR:
================================================================================

OBRIGATÓRIO:

  ✓ Código do projeto commitado no GitHub
    (vercel não faz deploy de código local)

  ✓ PostgreSQL em produção
    - URL formato: postgresql://user:pass@host:5432/dbname
    - Pode ser: Vercel Postgres, Railway, AWS RDS, etc
    - Salvar como: DATABASE_URL

  ✓ Redis em produção
    - URL formato: redis://:password@host:port
    - Pode ser: Redis Cloud, Railway, Upstash, etc
    - Salvar como: REDIS_URL

  ✓ NEXTAUTH_SECRET (gerador incluído)
    - Gerar: openssl rand -base64 32
    - Copiar para Vercel dashboard
    - NUNCA commitar para Git

OPCIONAL MAS RECOMENDADO:

  - Domínio customizado (você pode usar .vercel.app primeiro)
  - Certificado SSL (Vercel faz automático)
  - CDN global (Vercel faz automático)

================================================================================
VARIÁVEIS DE AMBIENTE (CRÍTICO!):
================================================================================

Você PRECISA configurar isto NO VERCEL DASHBOARD antes de fazer deploy:

OBRIGATÓRIAS:

  DATABASE_URL = postgresql://...
  REDIS_URL = redis://...
  NEXTAUTH_SECRET = (gerado com openssl rand -base64 32)
  NEXTAUTH_URL = https://seu-projeto.vercel.app ou seu-dominio.com
  APP_URL = https://seu-projeto.vercel.app ou seu-dominio.com
  NEXT_PUBLIC_BASE_URL = https://seu-projeto.vercel.app ou seu-dominio.com

OPCIONAIS (se uses):

  OPENAI_API_KEY = sk-...
  BIRD_API_KEY = ...
  BREVO_API_KEY = ...
  RESEND_API_KEY = ...
  SMTP_HOST, SMTP_USER, MAIL_FROM = ...
  IMAP_HOST, IMAP_USER = ...

================================================================================
TROUBLESHOOTING (3 CENÁRIOS COMUNS):
================================================================================

1. BUILD FALHA:
   $ vercel logs --tail
   (procure pela mensagem de erro exata)
   → Comum: DATABASE_URL inválida, NEXTAUTH_SECRET não definido

2. SITE MOSTRA ERRO 500:
   - Verificar DATABASE_URL está acessível
   - Verificar NEXTAUTH_SECRET foi definido
   - Verificar NEXTAUTH_URL = seu domínio

3. QUERO REVERTER DEPLOY:
   - Ir a https://vercel.com/dashboard
   - Deployments → clicar no anterior → "Promote to Production"

Ver DEPLOY_VERCEL_GUIA.md seção 6 para mais cenários

PRÓXIMAS ETAPAS PÓS-DEPLOY:
================================================================================

1. Configure domínio custom (opcional):
   - Vercel dashboard → Settings → Domains
   - Apontaur nameservers no seu registrador de domínio

2. Habilite Analytics (recomendado):
   - Vercel dashboard → Analytics
   - Veja performance, erros, usage em tempo real

3. Configure Webhooks (futuro):
   - Vercel dashboard → Settings → Webhooks
   - Integre com Slack, Discord, etc para notificações

4. Monitore Deployments:
   - Cada push → deploy automático
   - Vercel dashboard → Deployments para histórico
