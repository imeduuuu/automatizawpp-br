============================================================
INSTRUÇÕES DE USO
============================================================

1. CONFIGURAR AMBIENTE
   a) Gere um token seguro:
      openssl rand -base64 32
   
   b) Adicione ao .env:
      PUBLIC_DASHBOARD_TOKEN="seu_token_gerado"

2. TESTAR LOCALMENTE
   a) npm run dev
   b) Use test-public-api.sh ou cURL:
      TOKEN="seu_token"
      curl -H "Authorization: Bearer $TOKEN" \
        http://localhost:3000/api/public/leads

3. INTEGRAR NO FRONTEND
   a) Siga as instruções em FRONTEND_INTEGRATION.md
   b) Use hook usePublicAPI() nos componentes React
   c) Exemplos prontos: LeadsTable, Dashboard, ConversationsFeed

4. DEPLOYAR
   a) Configure PUBLIC_DASHBOARD_TOKEN em produção
   b) Deploy via Vercel
   c) Teste endpoints em produção
   d) Integre no automatizawpp.com

============================================================
ENDPOINTS RESUMO
============================================================

GET /api/public/leads
  Params: status, score, page (default: 1), limit (default: 20, max: 100)
  Restrições: status != NEW, createdAt >= 30 dias
  Auth: Bearer token

GET /api/public/conversations
  Params: channel, minScore, page, limit
  Restrições: Apenas conversas de leads pagos
  Auth: Bearer token

GET /api/public/analytics
  Params: Nenhum
  Retorna: 6 KPIs (totalLeads, emailsSent, callsCompleted, averageScore,
                   conversionRate, responseTime)
  Auth: Bearer token

SUPORTE
============================================================

Erros comuns:

1. "Invalid or missing token"
   → Verifique PUBLIC_DASHBOARD_TOKEN no .env
   → Verifique header: Authorization: Bearer <token>

2. Dados vazios
   → Verifique: há leads com status != NEW?
   → Verifique: há leads com createdAt >= 30 dias?

3. Erro 500
   → Verifique logs do servidor
   → Confirme que banco de dados está conectado

Para mais detalhes, veja TESTING_GUIDE.md
