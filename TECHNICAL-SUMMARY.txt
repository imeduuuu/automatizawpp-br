REQUISITOS DO SCRIPT
====================

Solicitação Original:
───────────────────
✓ Detecta plataforma disponível (Vercel, GitHub Pages, Firebase, DigitalOcean)
✓ Faz deploy automático
✓ Verifica se funcionou
✓ Envia notificação de sucesso/erro
✓ Executável com um único comando: './auto-deploy.sh'
✓ Tratamento de erros e fallbacks

├─ Build Functions (80 linhas)
│  ├─ build_project()

   ├─ Trap para Ctrl+C

   ├─ Summary formatado

Plataformas Suportadas
──────────────────────

VERCEL (Serverless)
├─ Detecção: vercel.json ou .vercelignore
├─ Verificação: vercel CLI ou $VERCEL_TOKEN
├─ Deploy: vercel deploy --prod
├─ URL: Extraída de output ou construída
└─ Tempo: 10-30s

FIREBASE (Serverless + Backend)
├─ Detecção: .firebaserc existência
├─ Verificação: firebase CLI ou $FIREBASE_TOKEN
├─ Deploy: firebase deploy --force
├─ URL: Construída de .firebaserc
└─ Tempo: 1-3 minutos

GITHUB PAGES (Git-based)
├─ Detecção: Git remote origin contém github.com
├─ Verificação: Git configurado
├─ Deploy: git subtree push origin gh-pages
├─ Cria branch gh-pages automaticamente
├─ URL: Construída de owner/repo
└─ Tempo: 20-60s

DIGITALOCEAN (VPS + App Platform)
├─ Detecção: app.yaml existência
├─ Verificação: doctl CLI e $DIGITALOCEAN_ACCESS_TOKEN
├─ Deploy: doctl apps update --spec app.yaml
├─ URL: Construída de app.yaml
└─ Tempo: 2-5 minutos

DOCKER (Container)
├─ Detecção: Dockerfile existência
├─ Verificação: Docker daemon rodando
├─ Build: docker build -t app:latest
├─ Deploy: docker-compose up -d (se existir)
├─ Health check: localhost (não remote)
└─ Tempo: 2-5 minutos


Notificações
────────────

EMAIL
├─ Método: Comando 'mail' nativo
├─ Conteúdo: Status, plataforma, URLs, timestamps
├─ Subject: [AutoDeploy] Deploy SUCESSO/FALHOU
├─ Fallback: Warning se 'mail' não disponível
└─ Plataformas: macOS, Linux (com postfix/sendmail)

SLACK
├─ Método: Webhook HTTP POST
├─ Content-Type: application/json
├─ Payload: Attachments com campos estruturados
├─ Cores: green (sucesso), red (erro)
├─ Campos: status, plataforma, URLs, timestamp
└─ Retentativas: 1x (não implementado)

WEBHOOK CUSTOMIZADO
├─ Método: HTTP POST JSON
├─ Payload: status, platform, build_success, deploy_success, health_check_success, deploy_url, timestamp, message
├─ Content-Type: application/json
├─ Segurança: HTTPS recomendado
└─ Seu servidor processa


Error Handling & Fallbacks
─────────────────────────

Validação de Arquivos
├─ package.json obrigatório?
│  └─ Não, pula build se não existir
├─ vercel.json, .firebaserc, etc?
│  └─ Pula plataforma se arquivo não existe
└─ Resultado: Deploy pode continuar com fallbacks

Build Falhas
├─ npm install falha?
│  └─ Script para
├─ npm build não existe?
│  └─ Pula etapa (BUILD_SUCCESS=1 mesmo assim)
├─ npm build falha?
│  └─ Script para com erro (retorna 1)
└─ Retry: Não implementado (user deve re-run)

Deploy Falhas
├─ CLI não instalada?
│  └─ Pula para próxima plataforma (|| true)
├─ Token inválido?
│  └─ Falha com mensagem clara
├─ Deploy falha?
│  └─ Script para e notifica erro
└─ Múltiplas plataformas: Tenta próxima

Health Check Falhas
├─ Timeout?
│  └─ Retry com 10s de aguardo
├─ HTTP non-200?
│  └─ Tenta próximo endpoint
├─ Todas falham?
│  └─ Warning log, deploy continua (não falha)
└─ Max retries: 5

Notificação Falhas
├─ Email: 'mail' não existe?
│  └─ Warning, continua
├─ Slack: Webhook falha?
│  └─ Error log, continua
├─ Webhook: conexão falha?
│  └─ Error log, continua
└─ Script não falha por notificação

Interrupção (Ctrl+C)
├─ Trap: INT TERM
├─ Ação: Cleanup minimal
├─ Logging: Registra interrupção
├─ Notificação: Envia "FALHOU" status
└─ Exit code: 1

├─ Visibilidade: Não exibidos em output/logs
└─ Best practice: Rotate regularmente

Logs
├─ Contém: Apenas status e URLs (não tokens)
├─ Acesso: /tmp (local user only)
├─ Retenção: Até reboot
├─ Cleanup: Manual (não auto-delete)

├─ Sempre: HTTPS (não HTTP)
