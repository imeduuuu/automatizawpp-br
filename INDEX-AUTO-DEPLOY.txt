FLUXOS DE TRABALHO COMUNS
=========================

CENÁRIO 1: "Quero deployar agora"
──────────────────────────────────
1. Verificar se projeto tem vercel.json, .firebaserc ou Dockerfile
2. Exportar token: export VERCEL_TOKEN=...
3. Rodar: ./auto-deploy.sh
4. Email notification: ./auto-deploy.sh --notify-email=seu@email.com

Tempo: 3 minutos

CENÁRIO 2: "Quero entender como funciona"
────────────────────────────────────────
1. Ler AUTO-DEPLOY-QUICK-REFERENCE.txt (5 min)
2. Executar exemplo: bash AUTO-DEPLOY-EXAMPLES.sh (escolher opção 1)
3. Ler AUTO-DEPLOY-README.md (20 min)
4. Rodar teste: bash test-auto-deploy.sh

Tempo: 30 minutos

CENÁRIO 3: "Quero usar em CI/CD (GitHub Actions)"
──────────────────────────────────────────────────
1. Criar .github/workflows/deploy.yml
2. Adicionar secrets: VERCEL_TOKEN, FIREBASE_TOKEN, etc
3. Fazer push do auto-deploy.sh
4. Criar .env.example com variáveis necessárias
5. Push para main → deploy automático!

Ver setup: bash AUTO-DEPLOY-EXAMPLES.sh → opção 6

Tempo: 10 minutos

CENÁRIO 4: "Tenho um Docker local e quero deployar"
────────────────────────────────────────────────────
1. Ter Dockerfile e docker-compose.prod.yml na raiz
2. Rodar: ./auto-deploy.sh --platform=docker
3. Verificar: curl http://localhost:3000

Ver setup: bash AUTO-DEPLOY-EXAMPLES.sh → opção 5

Tempo: 5 minutos

CENÁRIO 5: "Quero GitHub Pages para site estático"
───────────────────────────────────────────────────
1. npm run build (gera dist/, build/ ou out/)
2. ./auto-deploy.sh --platform=github-pages
3. Site automático em: https://seu-usuario.github.io/seu-repo

Tempo: 2 minutos

CENÁRIO 6: "Quero Slack notification quando deploy terminar"
────────────────────────────────────────────────────────────
1. Criar webhook em: https://api.slack.com/messaging/webhooks
2. Copiar URL do webhook
3. ./auto-deploy.sh --notify-slack=https://hooks.slack.com/services/...

Tempo: 3 minutos

FAQ RÁPIDO
==========

P: Como paro o deploy?
R: Ctrl+C — script detecta e limpa tudo

P: Deploy falhou, o que fazer?
R: cat /tmp/auto-deploy-*.log (veja a causa do erro)

P: Posso usar múltiplas notificações?
R: Não (por design simples), mas webhook pode replicar para várias plataformas

P: Preciso de Docker instalado para tudo?
R: Só se usar --platform=docker. Outras plataformas usam sua própria CLI

P: Posso customizar o script?
R: Sim! Veja AUTO-DEPLOY-README.md seção "Customização"

P: Quanto tempo leva?
R: Tipicamente 1-5 minutos total (build + deploy + health check)

P: Posso rodar sem notificações?
R: Sim, sem --notify-* executa sem enviar notificações

P: Health check falha, o que fazer?
R: Implementar um endpoint /health, /api/health, /status ou /ping
   Ou ignorar (deploy continua, apenas warning)

P: Como usar em produção?
R: 1) Testar localmente primeiro
   2) Adicionar a CI/CD (GitHub Actions)
   3) Usar secrets para tokens
   4) Configurar notifications
